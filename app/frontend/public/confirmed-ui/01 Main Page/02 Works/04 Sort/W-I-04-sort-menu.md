# W-I-04 Sort Menu

日期：2026-06-20

## 当前状态

讨论稿 v1。用于设计作品集页面顶部 `最近更新` 排序按钮被点击后的页内菜单。

## 交互目标

用户点击顶部 `最近更新` 右侧的向下箭头后，不跳转页面，也不打开大型弹窗，而是在按钮下方丝滑向下滑出小型纸感菜单。排序只改变中间作品卡片顺序，不改变当前筛选条件。

## v1 视觉规则

- 排序按钮进入激活态：浅纸色底、低亮棕色描边，箭头保持向下，表示这是下拉入口。
- 菜单位于排序按钮正下方，宽度与 `最近更新` 按钮一致。
- 打开状态下，`最近更新` 按钮与菜单应被视为同一个连体控件：触发按钮是同一外壳的顶部区域，菜单从外壳下半部分延展出来。
- 菜单顶部与按钮底部连接，不保留间隙，也不绘制独立分割线。
- 外壳只保留顶部圆角和底部圆角，中间连接处不保留按钮底部圆角，让展开关系更丝滑。
- 菜单保留羊皮纸质感和柔和阴影，不使用整页遮罩。
- 删除菜单上方的小白色三角箭头。
- 当前选中的 `最近更新` 使用浅纸色行、左侧细条和浅棕描边。
- 每个选项只保留主标题，不显示 `新→旧`、`A→Z`、`高→低` 等方向文字。
- 菜单层级高于卡片，但不压暗整个页面。

## 动效规则

- 打开时，菜单从 `最近更新` 按钮底部向下滑出。
- 展开起点应贴合按钮底部，避免先出现空隙再落下；视觉上像从按钮内部延展出来，而不是一个独立浮层贴在下面。
- 建议动画时长 180-220ms。
- 建议 easing 使用 `cubic-bezier(0.22, 1, 0.36, 1)`，形成轻快但不弹跳的滑出感。
- 菜单展开过程中透明度从 0 到 1，纵向位移从 `-8px` 到 `0`。
- 关闭时反向收回到按钮下方。

## 排序选项

| 显示文案 | 排序字段 | 默认方向 | 说明 |
| --- | --- | --- | --- |
| 最近更新 | `updatedAt` | `desc` | 最近编辑过的作品优先 |
| 最近打开 | `lastOpenedAt` | `desc` | 最近进入工作台的作品优先 |
| 创建时间 | `createdAt` | `desc` | 最新创建的作品优先 |
| 标题 | `title` | `asc` | 按作品标题字母或拼音顺序 |
| 完成度 | `progress` | `desc` | 完成度更高的作品优先 |

## 交互规则

| 操作 | 页面反应 |
| --- | --- |
| 点击 `最近更新` 右侧向下箭头 | 菜单从按钮下方滑出 |
| 点击当前选项 | 保持当前排序并关闭菜单 |
| 点击其它选项 | 更新排序字段，重新排列卡片，关闭菜单 |
| 再次点击排序按钮 | 关闭菜单 |
| 点击页面其它区域 | 关闭菜单 |
| 按 `Escape` | 关闭菜单 |

## 数据行为

```ts
type WorksSortKey =
  | "updatedAt"
  | "lastOpenedAt"
  | "createdAt"
  | "title"
  | "progress";

type WorksSortDirection = "asc" | "desc";

interface WorksSortState {
  isOpen: boolean;
  key: WorksSortKey;
  direction: WorksSortDirection;
}
```

排序逻辑与筛选逻辑组合：

```ts
const visibleStories = sortStories(
  filterStories(stories, filterState),
  sortState,
);
```

## 实现注意

- 排序不应清除当前状态筛选、题材筛选或搜索关键词。
- 排序后如果当前选中作品仍在可见列表中，右侧预览保持该作品。
- 如果当前选中作品因为其它筛选不在列表中，再回退到排序后第一部作品。
- 菜单宽度固定为 `最近更新` 按钮宽度，避免文字变化导致位置跳动。
- 打开态建议用一个整体 Popover Shell 承载触发按钮和菜单内容，按钮底部不能保留圆角、底部描边或独立阴影，否则连接处会出现硬线。
- 如果后续需要升序/降序切换，可在同一选项再次点击时切换方向；当前 v1 先不做二级切换。
