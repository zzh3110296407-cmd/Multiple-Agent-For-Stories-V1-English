# 08 Story Setup UI Audit / 2026-07-03

## 审查范围

本次审查覆盖 `08 Story Setup` 已生成的 6 个 UI 原型：

1. `01 Setup Entry`：设定入口。
2. `02 Generating`：生成中。
3. `03 Draft Review`：草案审阅。
4. `04 Missing Information Handling`：缺失信息处理。
5. `05 Review And Decision`：审查与决策。
6. `06 Handoff Initialization`：交接与初始化。

审查目标：确认 08 故事设定流程从“输入设定构想”到“创建交接包并初始化当前项目”的信息顺序、界面层级、交互路径和安全边界一致，不出现旧版 UI 信息、接口字段泄露、不可用交互或明显响应式缺陷。

## 已发现并修复的问题

1. 阶段条不一致。
   - 原问题：01-06 页的顶部阶段条文案和顺序不统一，有的页面把补充信息、草案审阅和交接顺序拆散。
   - 修复：统一为 `设定入口 / 生成草案 / 审阅补充 / 决策交接`，并让当前页高亮对应宏阶段。

2. 用户界面出现英文或接口痕迹。
   - 原问题：01 页出现 `SETUP PROMPT`、02 页出现 `DRAFT FORMING`，03 页出现 `来源字段 world_canvas_draft_suggestion`。
   - 修复：改为中文用户文案；03 页改为 `草案来源`，展示 `世界画布草案 / 角色方向草案 / Framework 建议草案 / 章节路线草案`。

3. 第 05 页决定状态不联动。
   - 原问题：选择并记录 `请求修订` 后，页面局部状态已是修订请求，但顶部状态卡仍显示 `通过`。
   - 修复：顶部状态卡、右侧草案摘要、页面状态和决定记录 ID 现在随决定类型联动：
     - 确认交接：通过 / 准备交接 / 已确认交接。
     - 请求修订：需修订 / 不进入交接 / 修订请求。
     - 延后决定：暂缓 / 等待后续审阅 / 延后决定。
     - 拒绝草案：已拒绝 / 不进入交接 / 已拒绝。
